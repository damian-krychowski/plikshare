How this works

Storages below use full encryption. When you unlock one, your password is sent to the server over HTTPS just once. The server uses it to derive the key that protects your files and then discards the password — it is never stored, never written to disk, and never logged.

That key is then sealed inside an encrypted session cookie that only this server can read. For the rest of your session, the browser sends that cookie with each request so the server can decrypt files on the fly.

Locking a storage simply discards the cookie. Your files stay encrypted at rest, and you will be asked for the password again the next time you want to use the storage.
